# GSTU report rules and cited standards

This summary is based on the local methodical guide `Оформление дипломных работ.pdf` from the workspace. It is a diploma-work guide, so the formatting rules below are used as the working GSTU ruleset for similar academic documents unless the user provides a narrower document-specific guide.

## Standards explicitly cited in the guide

- `ГОСТ 7.32-2001` - report structure and general research-report basis.
- `ГОСТ 7.1-2003` - bibliographic description and source list formatting.
- `СТБ 6.38-2004` - cited by the guide as a normative basis.
- `ГОСТ 2.105-95` - general requirements for text documents.
- `ГОСТ 8.417-2002` - units of physical quantities.
- `ГОСТ 2.316-2008` - accepted abbreviations.
- `ГОСТ 19.401-78` - program text formatting.
- `ГОСТ 19.503-79` - system programmer manual.
- `ГОСТ 19.504-79` - programmer manual.
- `ГОСТ 19.505-79` - operator or user manual.
- `РД 50-34.698-90` - recommended basis for user guide structure.

## Page layout and basic text formatting

- Paper format: `A4`, one-sided printing.
- Margins: top `20 mm`, bottom `20 mm`, left `30 mm`, right `10 mm`.
- Font: `Times New Roman`, `14 pt`.
- Line spacing: about `40 +/- 3` lines per page; in Word the guide recommends single spacing or multiplier `1.1`.
- Alignment: justified.
- Automatic hyphenation: allowed in body text, not in headings.
- First-line indent: `1.25 cm`.
- Font color: black, uniform through the whole document.
- Latin symbols in text: italic.

## Pagination

- Page numbers use Arabic numerals.
- Number position: bottom right.
- Font for page number: `Times New Roman`, `12 pt`.
- Title page counts as page 1 but is not numbered.
- Abstract, assignment, resume, and the first content sheet are included in total count but not numbered visibly.
- Visible numbering starts from the second content sheet, or from the abbreviations list, or from the introduction if earlier elements fit on one sheet.
- Appendices are included in the common pagination.

## Headings and section numbering

- Main text is divided into sections, subsections, and points.
- Sections: Arabic numerals, no trailing dot, from paragraph indent.
- Subsections: `section.subsection`, no trailing dot.
- Points: `section.subsection.point`.
- Sections and subsections must have headings; points usually do not.
- Section headings: uppercase letters, bold, from paragraph indent.
- Subsection headings: sentence case, bold, from paragraph indent.
- Point numbers: bold italic.
- No underline in headings.
- No period at the end of headings.
- No hyphenation inside headings.
- Every section starts on a new page.
- Keep at least some body text with a subsection heading on the same page.

## Academic style and wording

- Use scientific and technical terminology established by standards or accepted literature.
- Do not use colloquialisms, professional slang, arbitrary word formation, or interchangeable synonyms for the same concept.
- Do not shorten words except by accepted standard rules.
- Use standardized units per `ГОСТ 8.417-2002`.
- Write numbers with units in digits.
- Write numbers from one to nine in words when no unit follows.
- In requirements, use forms like `должен`, `следует`, `необходимо`, `не допускается`.

## Contents

- Contents must include introduction, abbreviations list if present, section and subsection titles, conclusion, source list, and appendices.
- Entries align vertically.
- Subsection entries are indented by one paragraph indent relative to sections.
- Dot leaders run to the page number.
- Appendix entries include the appendix letter and its title.

## Figures

- Place a figure immediately after the first textual reference or on the next page.
- All figures must be referenced in text.
- Number figures within each section: `Рисунок 3.4`.
- Caption is centered under the figure.
- Caption and figure are separated by a blank line.
- If a figure continues on later pages, keep the title on the first page and mark later sheets accordingly.
- In appendices use numbering like `Рисунок А.2`.

## Formulas

- The guide recommends `Equation 3.0`, but any editor is acceptable if the visual rules match.
- Latin variables: italic.
- Greek symbols: upright.
- Formulas are separated from text by blank lines.
- Explain symbols directly under the formula.
- The first explanation line starts with `где` and no colon.
- Number formulas within each section: `(3.1)`.
- Put the number at the right margin.
- Refer to formulas in text explicitly, for example `по формуле (2.1)`.

## Tables

- Place a table after the first textual reference or on the next page.
- All tables must be referenced in text.
- Table title goes above the table, left aligned from paragraph indent.
- Number tables within each section: `Таблица 4.2`.
- Titles should be concise and informative.
- Text above and below the table is separated by a blank line.
- Column headings use capital letters; subheadings use lowercase if they continue the heading meaning.
- No period in headings or subheadings.
- The `No.` column should not be added as a default column.
- On continuation pages use `Продолжение таблицы ...` or an equivalent automated continuation style.
- In appendices use numbering like `Таблица В.1`.

## Source list and in-text citations

- Source list title: `Список использованных источников`.
- Place it before appendices.
- Preferred ordering: by first appearance in the text.
- Alternative allowed: alphabetical order.
- Use Arabic numerals with a period and paragraph indent.
- Include only sources that are actually cited.
- In-text citations use square brackets, for example `[1]`.
- For large sources, page or figure уточнение is allowed, for example `[1, с. 386]`.
- Internet sources should be official resources of organizations, authorities, or official language or technology references where possible.
- Bibliographic formatting follows `ГОСТ 7.1-2003`.

## Appendices

- Appendices may be mandatory, recommended, or reference.
- Every appendix starts on a new page.
- Top center: `ПРИЛОЖЕНИЕ` plus its letter.
- Next line in parentheses: `обязательное`, `рекомендуемое`, or `справочное`.
- Next line: appendix title, centered.
- Appendices are ordered by first reference in the text.
- Every appendix must be referenced in the main text.
- Allowed appendix letters: Russian capital letters starting from `А`, excluding disallowed letters noted by the guide.
- Reduced font `10-12 pt` is allowed inside appendices if readability remains acceptable.
- Numbering of figures, tables, and formulas in appendices replaces the section number with the appendix letter.

## Program text and software documents

- Program code should be placed in appendices, not in the main text.
- For code appendices the guide allows `Times` `10-12 pt`, single spacing.
- Small code fragments up to about half a page may appear in the main text as figures if they are needed for explanation.
- If the report includes separate manuals, the guide points to:
  - `ГОСТ 19.503-79` for the system programmer manual;
  - `ГОСТ 19.504-79` for the programmer manual;
  - `ГОСТ 19.505-79` and `РД 50-34.698-90` for the user manual.

## Safe adaptation for GSTU reports

Use the formatting rules above directly. For structure, adapt them to the requested document type. For a report, this default structure is safe:

1. Title page.
2. Assignment or individual task, if required.
3. Contents.
4. Abbreviations, if needed.
5. Introduction.
6. Description of the organization or project context.
7. Analysis of the subject area and tools.
8. Description of completed work.
9. Results and evaluation.
10. Conclusion.
11. Source list.
12. Appendices.
